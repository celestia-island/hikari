# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-02-05

### Added
- Initial release of Hikari UI framework
- 500+ Chinese traditional colors in palette system
- Complete component library with 60+ components
- Animation system with 30+ easing functions
- Icon system with 1000+ Material Design Icons
- Theme system supporting light/dark themes
- SSR support with render service
- Build system with SCSS compilation
- Comprehensive documentation and examples

### Features
- **hikari-palette**: Chinese traditional color system with utility classes
- **hikari-theme**: Theme provider and CSS variable management
- **hikari-icons**: Icon component with Lucide and MDI icons
- **hikari-animation**: Declarative animation system inspired by GSAP
- **hikari-components**: Complete UI component library
  - Basic components: Button, Input, Card, Badge, etc.
  - Form components: Select, Checkbox, Radio, Switch, etc.
  - Navigation components: Menu, Tabs, Breadcrumb, etc.
  - Data components: Table, Tree, Pagination, etc.
  - Feedback components: Alert, Toast, Tooltip, etc.
- **hikari-extra-components**: Advanced interactive components
  - Collapsible panels with animations
  - Drag and drop layer system
  - Zoom controls with keyboard shortcuts
  - Rich text editor
  - Audio waveform visualization
  - Node graph editor
- **hikari-render-service**: SSR and static asset serving
- **hikari-builder**: Build system with SCSS compilation and code generation

### Design
- Arknights-inspired flat design language
- FUI (Fantasy User Interface) sci-fi aesthetics
- Chinese traditional color palette integration
- Clean typography and spacing system

### Technical
- Rust + Dioxus framework
- WASM compilation support
- Axum server integration
- Grass SCSS compiler
- Parallel E2E testing framework
- Comprehensive test coverage (100% unit tests)

### Documentation
- Complete API documentation
- Interactive examples
- Architecture documentation
- Development guidelines
- Component showcase website
