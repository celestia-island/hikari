# i18n Internationalisation

Le système d'internationalisation de Hikari prend en charge le changement de plusieurs langues avec un support complet RTL (Right-to-Left).

## Langues Supportées

| Langue | Code | Direction |
|--------|------|-----------|
| Anglais | `en-US` | LTR |
| Chinois Simplifié | `zh-CHS` | LTR |
| Chinois Traditionnel | `zh-CHT` | LTR |
| Français | `fr-FR` | LTR |
| Russe | `ru-RU` | LTR |
| Espagnol | `es-ES` | LTR |
| Arabe | `ar-SA` | **RTL** |
| Japonais | `ja-JP` | LTR |
| Coréen | `ko-KR` | LTR |

## Utilisation de Base

```hikari
rsx! {
    div { style: "padding:1rem;color:#999;", "Component preview: pages/system/i18n#basic" }
}
```

## Changement de Langue

```hikari
rsx! {
    div { style: "padding:1rem;color:#999;", "Component preview: pages/system/i18n#switch" }
}
```

## Support RTL

Pour les langues RTL comme l'arabe, le `I18nProvider` définit automatiquement `dir="rtl"`:

```rust
use tairitsu_web::i18n::{I18nProvider, context::Language};

fn App() -> Element {
    rsx! {
        I18nProvider {
            language: Language::Arabic,  // RTL automatique
            toml_content: AR_SA_TOML,
            YourApp {}
        }
    }
}
```

Les composants de layout s'adaptent automatiquement à la direction RTL.

## API

| Propriété | Description | Type | Défaut |
|-----------|-------------|------|--------|
| language | Langue actuelle | Language | Anglais |
| toml_content | Contenu de traduction | &str | - |
| direction | Direction du layout (ThemeProvider) | "ltr" \| "rtl" | "ltr" |
